# 面向AI Agent桌面工业软件架构重构

> 我们正站在一个转折点上：软件的“用户”正在从“人类”扩展为“人类+AI Agent”。未来，一个软件如果无法被AI高效调用，将在智能经济时代被边缘化。

这篇文章探讨的是当`AI Agent`需要像人类专家一样操作CAD、仿真软件、数据分析软件时，我们的软件架构该如何进化？

`AI Agent`目前最好的应用场景是编程环境，编译器都可以支持命令行执行，AI Agent可以很轻松的调用这些**工具**，但工业软件设计之初就是为人类提供一个大型可视化工具，它要面对的场景太复杂，无法一句cli命令就完成工作，导致现在的工业软件重GUI交互，而少有命令提供，这样导致未来`AI Agent`很难调用工业软件来执行复杂的工作，因为很多软件设计的时候就不会考虑命令行

互联网软件有天然的优势，天然的前后端分离让实现一个MCP服务变得更容易。当然它需要把现阶段琐碎的api接口内容改为更像人，多个接口合并为一个大接口会更适合`AI Agent`“阅读”

## 面向AI的接口设计

从“数据接口”到“能力接口”

当前80%的软件都是数据的呈现，尤其以互联网软件为例，就是对数据库的CURD，因此，现在软件架构设计思考的是如何把数据在视图中呈现，典型的MVC模式，前端-后端-数据库的三层架构，未来面向AI Agent，现有架构要如何调整能让AI更好的感知和操作，是现在需要思考的问题，尤其桌面软件，很多缺少前后端分离思想设计的软件，如何改造如何把`工具`和`接口`暴露给AI是未来软件架构需要重点思考的方向

## 拥抱AI原生时代

未来的软件竞争，不仅是功能的竞争，更是被AI理解和调用的能力的竞争。当一个用户对着手机或电脑说“帮我处理一下这件事”时，你的软件是否能成为`AI Agent`的首选，这可能决定了你软件的占有率
