# 命令行参数

本文档介绍 DAWorkBench 的命令行参数使用方法，通过命令行可实现自动化数据导入和工程打开。

DAWorkBench 支持命令行参数，通过命令行可以控制程序启动行为，实现自动化数据导入和工程打开等功能。

## 主要功能特性

**特性**

- ✅ **工程文件打开**：直接通过命令行参数打开指定的 `.dapro` 工程文件
- ✅ **数据文件导入**：启动时自动导入 CSV、Excel 等格式数据文件
- ✅ **版本信息显示**：快速查看程序版本号
- ✅ **帮助信息显示**：查看所有可用命令行参数

## 参数一览

程序支持的命令行参数如下：

| 参数 | 简写 | 说明 |
|------|------|------|
| `--help` | `-h` | 显示帮助信息并退出 |
| `--version` | `-v` | 显示程序版本号并退出 |
| `[project]` | 无 | 打开指定的工程文件（位置参数） |
| `--import-data <path>` | 无 | 导入数据文件，支持多次使用 |

## 使用方法

### 打开工程文件

直接在命令行指定 `.dapro` 工程文件路径，程序启动后自动加载该工程。这是最常用的启动方式，适用于已有项目继续编辑：

```shell
# 打开指定工程文件
# .dapro 文件是 ZIP 格式的压缩包，包含工作流、数据和图表配置
DAWorkBench.exe "D:\projects\my-project.dapro"
```

!!! tip "路径引号"
    如果路径包含空格（如 `D:\My Projects\data.dapro`），需要使用双引号包裹路径。

### 导入数据文件

使用 `--import-data` 参数在启动时导入数据文件，适用于新项目或需要补充数据的场景。支持的数据格式如下：

| 格式 | 扩展名 | 说明 |
|------|--------|------|
| CSV | `.csv` | 逗号分隔值文件，最常用的数据格式 |
| Excel | `.xlsx` / `.xls` | Excel 工作簿，支持多工作表 |
| 文本文件 | `.txt` | 文本格式数据，自动检测分隔符 |
| Pickle | `.pkl` | Python 序列化数据，保留数据类型 |

**导入单个数据文件：**

以下命令导入单个 CSV 或 Excel 文件。程序启动后数据自动加载到数据管理器：

```shell
# 导入单个 CSV 文件
# CSV 文件使用逗号分隔，支持 UTF-8 和 GBK 编码自动检测
DAWorkBench.exe --import-data "D:\data.csv"

# 导入 Excel 文件
# Excel 文件默认读取第一个工作表
DAWorkBench.exe --import-data "D:\data.xlsx"
```

**导入多个数据文件：**

多次使用 `--import-data` 参数可导入多个数据文件，每个文件独立加载：

```shell
# 同时导入多个数据文件
# 各文件以独立数据项形式出现在数据管理器中
DAWorkBench.exe --import-data "D:\data1.csv" --import-data "D:\data2.xlsx"
```

**同时打开工程并导入数据：**

以下命令打开已有工程并导入额外数据，适用于补充数据到现有项目：

```shell
# 打开工程文件并导入额外数据
# 工程数据和新导入数据同时可用
DAWorkBench.exe "D:\my-project.dapro" --import-data "D:\additional-data.csv"
```

### 显示版本号

使用 `--version` 参数快速查看程序版本信息，无需启动主界面：

```shell
# 显示版本号并退出
# 用于脚本中检查程序版本或诊断问题
DAWorkBench.exe --version
```

程序输出示例（版本号格式为 `主版本.次版本.修订号`）：

```
DAWorkBench version 1.0.0
```

### 显示帮助信息

使用 `--help` 参数查看所有可用命令行参数，输出完整的参数说明：

```shell
# 显示帮助信息并退出
# 列出所有参数及其用法说明
DAWorkBench.exe --help
```

## 常见使用场景

### 场景一：批量数据处理

通过命令行参数实现批量数据处理的自动化启动。下图展示了典型的批处理工作流程：

```mermaid
flowchart LR
    A[命令行启动] --> B[导入数据文件]
    B --> C[执行工作流]
    C --> D[导出结果]
```

上图展示了批处理的核心流程：通过命令行参数启动程序并导入数据，程序自动执行预定义的工作流处理数据，最后输出处理结果。这种方式适用于需要重复处理多组数据的场景。

命令示例（打开预配置的批处理工程并导入输入数据）：

```shell
# 执行批处理工作流
# batch-process.dapro 包含预定义的数据处理流程
# input-data.csv 是待处理的数据文件
DAWorkBench.exe "D:\batch-process.dapro" --import-data "D:\input-data.csv"
```

### 场景二：快捷启动指定工程

为常用工程创建 Windows 快捷方式，可直接打开目标工程无需手动选择文件。以下命令可设置为快捷方式的目标路径：

```shell
# Windows 快捷方式目标设置
# 快捷方式双击即可打开指定工程
"D:\Qt\bin\DAWorkBench.exe" "D:\projects\daily-analysis.dapro"
```

创建快捷方式的步骤：右键桌面 → 新建快捷方式 → 输入上述命令 → 完成。

## 注意事项

!!! warning "参数顺序"
    工程文件参数（位置参数）应放在其他参数之前，确保正确解析。

!!! info "数据格式兼容性"
    数据文件格式需与程序支持的解析器兼容。对于复杂格式的 Excel 文件，建议先在程序中手动导入测试。

!!! tip "自动化脚本"
    可将命令行参数与 Windows 批处理脚本或 PowerShell 结合，实现自动化数据处理流程。
